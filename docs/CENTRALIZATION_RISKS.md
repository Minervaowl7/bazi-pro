# 中心化风险评估

本文档记录 bazi-pro 中的中心化风险点、缓解措施和用户退出路径。

---

## 1. 风险概览

| 风险点 | 严重度 | 影响范围 | 缓解措施 |
|--------|--------|---------|---------|
| LLM API 依赖 | 中 | LLM 解读功能 | 可禁用、多供应商支持 |
| Redis 依赖 | 低 | 缓存/限流/任务存储 | 内存降级 |
| SQLite 单写入者 | 低 | 并发写入 | WAL 模式、单 worker |
| 单一部署实例 | 中 | 高可用 | 可水平扩展（需改造） |

---

## 2. LLM API 依赖

### 2.1 风险描述

bazi-pro 的 LLM 辅助解读功能依赖外部 OpenAI 兼容 API（OpenAI、DeepSeek、通义千问等）。

**依赖范围**：
- 命局总览生成
- 分维度解读
- 候选格局裁决
- 调候用神补充

**不依赖 LLM 的功能**：
- 核心计算（十神、藏干、旺衰、格局、用神）
- 古籍检索（BM25 本地索引）
- 叙述器（确定性文本生成）
- 神煞/宫位计算

### 2.2 缓解措施

**已实施**：
- LLM 功能可完全禁用：设置 `LLM_API_KEY=""`
- 支持多供应商：任何 OpenAI 兼容 API
- 超时控制：`LLM_TIMEOUT` 环境变量（默认 300s）
- 错误降级：LLM 调用失败时跳过，不影响核心功能

**可选改进**：
- 本地 LLM 支持（Ollama）
- LLM 响应缓存（相同输入不重复调用）

### 2.3 用户退出路径

```bash
# 禁用 LLM 功能
export LLM_API_KEY=""
# 或在 .env 文件中设置
LLM_API_KEY=
```

禁用后：
- 核心分析功能完全正常
- 叙述器文本正常生成
- LLM 相关字段返回空值

## 3. Redis 依赖

### 3.1 风险描述

Redis 用于：
- 分析结果缓存
- 速率限制计数器
- 任务存储（异步任务状态）

### 3.2 缓解措施

**已实施**：
- 所有 Redis 功能均有内存降级：
  - 缓存 → LRU 内存字典
  - 限流 → 内存滑动窗口
  - 任务存储 → 内存字典
- 降级状态通过 `/api/health` 暴露
- 降级时自动记录 WARNING 日志

**无 Redis 时的行为**：
- 功能完全正常
- 缓存容量有限（默认 128 条）
- 多实例间不共享状态
- 重启后缓存/任务状态丢失

### 3.3 用户退出路径

```bash
# 不设置 REDIS_URL 即可（默认无 Redis）
# 或清空
export REDIS_URL=""
```

## 4. SQLite 单写入者限制

### 4.1 风险描述

SQLite 使用文件锁，同一时间只允许一个写入者。这限制了：
- 多 worker 部署
- 高并发写入场景

### 4.2 缓解措施

**已实施**：
- WAL 模式：`PRAGMA journal_mode=WAL`（允许并发读）
- 强制单 worker：`server/app.py` 中 `workers=1`
- 异步 I/O：使用 `aiosqlite` 避免阻塞

**影响**：
- 并发读取正常
- 写入排队（通常 < 10ms）
- 适合中小规模部署

### 4.3 升级路径

如需更高并发：
1. 迁移到 PostgreSQL
2. 使用 Redis 作为主存储
3. 实现数据库连接池

---

## 5. 单实例部署

### 5.1 风险描述

默认部署为单实例，存在：
- 单点故障
- 无法水平扩展
- 维护时服务中断

### 5.2 缓解措施

**当前架构允许**：
- 进程管理器自动重启（systemd、supervisor）
- 健康检查端点（`/api/health`）
- 优雅关闭（lifespan 管理）

**高可用方案**（需改造）：
1. 多实例 + 负载均衡
2. 共享 Redis（缓存/限流/任务存储）
3. PostgreSQL 替代 SQLite
4. 容器编排（Kubernetes）

## 6. 数据隐私

### 6.1 数据流向

| 数据 | 存储位置 | 是否发送到外部 |
|------|---------|---------------|
| 八字输入 | SQLite（本地） | 否 |
| 分析结果 | SQLite + 缓存 | 否 |
| 古籍检索 | 本地文件 + BM25 索引 | 否 |
| LLM 提示词 | 无持久化 | 是（发送到 LLM API） |
| 用户 IP | 日志（本地） | 否 |

### 6.2 LLM 数据隐私

LLM 调用时发送的数据：
- 八字信息（四柱、日主、性别）
- 分析结果摘要
- 用户提问（聊天功能）

**不发送**：
- 用户身份信息
- IP 地址
- 历史记录

### 6.3 隐私保护措施

- LLM API Key 不记录到日志
- 用户输入不持久化到 LLM 提供商
- 可完全禁用 LLM 功能
- 本地部署时无数据外传

## 7. 供应商锁定

### 7.1 当前依赖

| 组件 | 供应商 | 锁定程度 | 替代方案 |
|------|--------|---------|---------|
| LLM API | OpenAI 兼容 | 低 | 任何 OpenAI 兼容 API |
| Redis | Redis Labs / 自托管 | 低 | 内存降级 |
| SQLite | 公有领域 | 无 | PostgreSQL、MySQL |
| jieba | 开源 | 无 | 其他中文分词器 |

### 7.2 降低锁定的措施

- LLM 使用标准 OpenAI API 格式
- 所有组件可本地部署
- 无强制云服务依赖
- 开源代码（MIT 许可证）
