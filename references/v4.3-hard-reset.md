# bazi-pro v4.3 Hard Reset — 从"HTML 报告生成器"升级为"命理分析产品界面"

> **核心原则：不要继续 patch CSS，要推倒 UI 生成方式。**
> **视觉概念：命理案卷 OS** — 案卷 · 裁决 · 证据链 · 命盘 · 朱批 · 印章 · 档案编号 · 可回放审查

## P0 — UI 架构重构

### 1. 拆掉 Python 大字符串 HTML → Jinja 模板架构
```
bazi_pro/ui/
  templates/
    dashboard.html.j2
    report.html.j2
    replay.html.j2
    components/
      hero.html.j2 / verdict_seal.html.j2
      four_pillars.html.j2 / verdict_card.html.j2
      evidence_card.html.j2 / relation_card.html.j2
      timeline.html.j2 / energy_ledger.html.j2
  static/
    tokens.css / layout.css / components.css / print.css / replay.js
view_model.py  # @dataclass DashboardVM, PillarVM, VerdictVM, etc.
```
- Python 只负责数据，不负责长 HTML 拼接
- Dashboard / Report / Replay 共用同一个 ViewModel
- 禁止从 Markdown 正则硬扒数据

### 2. ViewModel 替代正则硬扒
```python
@dataclass
class DashboardVM:
    pillars: list[PillarVM]
    summary: SummaryVM
    verdict: VerdictVM
    wuxing: WuxingVM
    evidence: list[EvidenceVM]
    relations: list[RelationVM]
    timeline: list[TraceStageVM]
```

## P1 — 视觉概念：命理案卷 OS

- 朱砂印章：显示"正格 / 不从 / 用木"
- 案卷编号：Run ID / Corpus Hash / Confidence
- 朱批边注：关键裁决理由
- 宣纸纹理：极轻
- 档案页签：Dashboard / Report / Replay
- 古籍引用卡：像书页摘录

### 3. Verdict Seal（品牌记忆点）
圆形/方形朱砂印，中心大字"正格"，副字"不从"，底部"用木 · 喜水金"

### 4. Dashboard 5 决策区（不堆模块）
```
1. Verdict Hero — 最终裁决
2. Why This Verdict — 三条关键依据
3. Energy Map — Element Balance Ledger（替代雷达图）
4. Evidence Dossier — 三张证据卡
5. Replay CTA — 查看裁决过程
```

### 5. Element Balance Ledger（替代雷达图）
每个五行显示：百分比 + 命理角色（印/比劫/食伤/财/官杀）+ 解读
像账簿，不像图表库。

### 6. 关系图谱 → 关系裁决卡
每条关系：影响 + 强度 + 裁决。小型 SVG 只做辅助，不让 SVG 当主角。
