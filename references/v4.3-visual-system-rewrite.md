# bazi-pro v4.3 Visual System Rewrite — Design Brief

> **核心诊断**：当前 UI 把 100% 数据全摊开，没有视觉层级，没有阅读路径。
> **目标气质**：Apple Health + Bloomberg Terminal + 东方命理卷轴
> **关键词**：克制 · 留白 · 层次 · 呼吸感 · 审计感 · 神秘但专业

---

## 产品气质

```
当前：玄学博客 + Grafana + 开源后台
目标：Apple Health + Bloomberg Terminal + 东方命理卷轴
```

## P0 — Dashboard 重做

### P0-1 删除 50% 边框
- card 取消默认边框，改用背景层级区分
- 仅 active/important card 保留描边
- 大量阴影 + 留白代替框线
- 参考：Linear / Notion / Stripe Dashboard

### P0-2 视觉层级（5 级 typography）
- Hero Number（72px）— 最终裁决
- Section Title（28px）
- Card Title（15px uppercase）
- Body（15px）
- Meta（12px）

### P0-3 Hero 区重做
当前顶部只展示四柱八字（太数据），应该展示：
```
丁火日主 · 七杀佩印格 · 偏旺 · 取木水 · 置信度 82%
```
四柱降级为 secondary info。

### P0-4 颜色系统重构
- 主色：Warm Ivory
- 强调色：Ancient Gold
- 危险：Muted Vermilion
- 成功：Jade Green
- 五行颜色：全部降 saturation 40%（现在太像儿童编程教育产品）

### P0-5 雷达图 → Horizontal Energy Flow
删掉 SVG 雷达图，改成 ultra-clean 的水平能量条（类似 Apple Health）。

## P1 — Evidence Inspector

### P1-1 改成"法庭裁决卡"
```
CLAIM: 日主偏旺，取木水
CONFIDENCE: 82%
SUPPORTING EVIDENCE: ...
COUNTER EVIDENCE: ...
FINAL DECISION: ...
```
视觉像 AI reasoning trace，不是 FAQ 折叠。

### P1-2 Evidence Timeline
```
① 旺衰判断 → ② 格局裁决 → ③ 顺向检索 → ④ 反事实检索 → ⑤ 用神决策
```
点击步骤，右侧显示 evidence（类似 OpenAI reasoning viewer）。

## P1 — 关系图谱

### P1-3 星盘式布局
```
日主居中
其他柱环绕
关系像星轨
```
- 冲 → 红色断裂轨道
- 合 → 金色柔线
- 刑 → 紫色虚线

## P1 — Report 重做

### P1-4 删除目录自动编号
改成卷式命名：
```
卷一 · 命局总论
卷二 · 旺衰裁决
卷三 · 格局与喜用
```

### P1-5 封面重做
```
BAZI ANALYSIS
丁火命局审查报告
Generated by bazi-pro v4.3
```
更像金融分析报告，不像淘宝算命 PDF。

## P2 — 神秘感

- 微弱粒子 / 缓慢 glow / hover 漂浮 / 数字 rolling（极轻，不赛博朋克）
- 开场动画：四柱淡入 → 五行流动 → 最终裁决浮现

## 架构建议

- 新建 design_tokens.css / typography.css / layout.css / evidence.css / timeline.css / report.css
- 不要继续 patch CSS

## 命名

v4.3 应叫：**The Oracle Trace** 或 **Judgement Engine**
