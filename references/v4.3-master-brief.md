# bazi-pro v4.3 UI/UX Redesign — Master Brief

> 目标：把当前 HTML 从"工程生成页面"重构为 **高级、克制、可读、可回放、可截图传播** 的命理分析产品页面。
> 参考问题文件：`bazi_dashboard_20020519_dashboard.html` / `bazi_report_20020519.html`

## 三种输出形态

```
1. Dashboard：给用户快速理解，一屏看结论
2. Report：给用户阅读和打印，像正式咨询报告
3. Replay Viewer：给用户复盘推理链，解释"为什么这么判"
```

不要把所有内容堆在一个页面里。

---

## P0 — 先修最丑、最影响信任

### 1. 重做 Dashboard 首屏
- Hero 区：日主 + 格局 + 主裁决 + 用神 + 喜忌 + 置信度 + 一句话摘要
- 首屏 ≤ 700px，不滚动即可看到核心结论
- 长术语不要挤在圆环卡片里

### 2. 删除所有"数据未提取"占位
- 不渲染缺失模块，或显示空状态卡片
- 页面不得出现"未提取""None""null""undefined""NaN"
- CI 增加 HTML smoke test

### 3. 修复 TOC 目录结构
- 重写 TOC 生成器，不要手拼 HTML
- heading stack 算法生成合法嵌套
- 不允许连续重复 `</ol></li>`

## P1 — 视觉系统

### 4. Design System: Neo Classical
```css
--bg: #f7f1e8;        --surface: #fffaf2;
--ink: #241a14;       --muted: #7a6a58;
--primary: #8a3b2a;   --gold: #b99a5b;
--wood: #5f8d72; --fire: #b85c4a; --earth: #b98b54;
--metal: #b7aa8b; --water: #4f7896;
--radius-lg: 24px; --radius-md: 16px;
```
- 五行色降饱和度，不刺眼
- Dashboard / Report / Replay 共用同一 token 文件

### 5. Dashboard → Bento Grid
```
┌──────────────────────────────┐
│ Hero 总结                     │
├──────────────┬───────────────┤
│ 四柱命盘      │ 用神裁决       │
├──────────────┼───────────────┤
│ 五行力量      │ 格局评分       │
├──────────────┴───────────────┤
│ Replay Timeline              │
├──────────────┬───────────────┤
│ 关系图谱      │ 证据链摘要      │
└──────────────┴───────────────┘
```
- CSS Grid，卡片尺寸有主次
- Evidence 和详细报告默认折叠

### 6. 重做四柱卡片
- 每柱显示：天干 + 地支 + 十神 + 五行 + 藏干比例条
- 日柱高亮但不刺眼

## P1 — 信息架构

### 7. Dashboard 只保留摘要级信息
- 移除完整正文，增加"查看完整报告"+"回放推理链"按钮
- Dashboard HTML 长度明显下降，控制在 2-3 屏

### 8. 证据链改成"摘要 + 展开详情"
- `<summary>` ≤ 28 个中文字符
- 长文本放进 body
- 每条 evidence 必须有 claim / confidence / basis / counter_evidence / decision

### 9. 关系图谱要讲人话
- 每条 edge 配 tooltip 或解释列表
- 显示"关系 → 影响 → 强度"
- 没有关系时不渲染空图

---

## P2 — Report 页面重构

### 10. Report 从"论文页"改成"咨询报告"
- 开头必须有 Executive Summary（5 条关键结论 + 风险提示 + 行动建议）
- 技术步骤放附录，不放正文最前面

### 11. 减少缩进和密度
- 去掉 text-indent: 2em
- 每大段前有小结卡片
- 表格不连续堆太多

### 12. 表格视觉升级
- 四柱 → 卡片式
- 神煞 → 分组（吉星/桃花/驿马/凶星/空亡）
- 检索结果 → Evidence Cards
- 大运 → 横向 timeline

---

## P2 — Replay Viewer (新增)

### 13. 新增 analysis_trace.json
```json
{
  "schema_version": "trace.v1",
  "summary": { "day_master": "丁火", "pattern": "...", "decision": "..." },
  "stages": [
    {"id": "precheck", "title": "五行快速预检", "status": "done"},
    {"id": "retrieval_dual_channel", "title": "双通道检索", ...},
    {"id": "final_decision", "title": "最终裁决", "confidence": 0.8}
  ]
}
```
- Dashboard 和 Replay Viewer 从同一个 trace JSON 读取

### 14. Replay Timeline UI
```
① 数据校验 → ② 五行预检 → ③ 旺衰判断 → ④ 格局筛查
→ ⑤ 双通道检索 → ⑥ 反事实比较 → ⑦ 喜用裁决 → ⑧ 最终结论
```
- 点击每步显示：输入 + 规则 + 证据 + 反证 + 输出 + 置信度

## 最终验收标准

```
Dashboard: 3 秒内知道核心结论
Report: 像正式咨询报告，不是 Markdown 转 HTML
Replay: 能逐步看到"规则 → 检索 → 反证 → 裁决"
截图：一张图看起来高级，不像调试页面
```

**方向：少一点"我把数据都展示给你看"，多一点"我帮你把结论讲清楚"。**
